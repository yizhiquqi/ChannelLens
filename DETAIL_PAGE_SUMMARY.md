## 🏗️ 组件架构

### 单一 PartnerDetailPage 组件的优势

```typescript
// 使用条件渲染，根据 entityType 选择不同的 Section 组件
export default function PartnerDetailPage({ channelId, onNavigate }) {
  const partner = mockPartners.find(p => p.id === channelId);

  return (
    <div>
      {/* 类型特异性 Sections */}
      {partner.entityType === 'company' && <CompanySection partner={partner} />}
      {partner.entityType === 'person' && <PersonSection partner={partner} />}
      {partner.entityType === 'team' && <TeamSection partner={partner} />}
      
      {/* 共享 Sections - 所有类型都显示 */}
      <SharedSections partner={partner} verifiedReviews={verifiedReviews} onNavigate={onNavigate} />
      
      {/* 右侧面板 - 所有类型都相同 */}
      <SidePanel partner={partner} verifiedReviews={verifiedReviews} onNavigate={onNavigate} />
    </div>
  );
}
```

### 子组件

- **CompanySection** - 公司相关信息展示
- **PersonSection** - 个人相关信息展示（+ 隐私保护）
- **TeamSection** - 团队相关信息展示
- **SharedSections** - 所有类型通用的 Sections
- **ReviewCard** - 单条评价卡片

**优点：**
- ✅ DRY 原则 - 避免代码重复
- ✅ 易于维护 - 修改共享逻辑只需改一个地方
- ✅ 易于扩展 - 增加新类型只需新增 Section 组件
- ✅ 清晰的职责分工 - 每个组件专注一个类型
